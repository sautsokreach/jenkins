What Is Jenkins & How It Works
▼

Jenkins is an open-source CI/CD automation server. It watches your code repository and automatically runs tasks (build, test, deploy) whenever code changes.

CI (Continuous Integration) = automatically build & test every commit.
CD (Continuous Delivery/Deployment) = automatically ship the result.

Core Concepts

TermMeaning
Job / PipelineA configured automation task
BuildOne execution of a job
StageA named phase inside a pipeline
StepA single command inside a stage
AgentThe machine that runs your build
NodeA Jenkins server/worker
WorkspaceThe directory where your code is checked out
ArtifactA file produced by a build (e.g., app.jar)

Jenkins Architecture

┌─────────────────────────────────────┐
│           Jenkins Controller         │
│  (schedules jobs, stores config)     │
└────────────────┬────────────────────┘
                 │ distributes work
       ┌─────────┴──────────┐
       ▼                    ▼
  Agent Node 1         Agent Node 2
  (Linux/Maven)        (Docker/Node)
Your Setup: Your Jenkinsfile uses agent any — Jenkins picks whatever agent is free.
02
Declarative vs Scripted Pipeline
▼

Declarative (recommended — what you have)

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'mvn clean package'
            }
        }
    }
}

Scripted (older, more flexible)

node {
    stage('Build') {
        sh 'mvn clean package'
    }
}
Rule of thumb: Use Declarative. Fall back to Scripted only when you need advanced logic inside a script {} block.
03
Anatomy of YOUR Jenkinsfile (Line by Line)
▼

Top-Level Structure

pipeline {          // Declares a Declarative Pipeline
    agent any         // Run on any available agent

    environment {     // Global env vars available in ALL stages
        NEXUS_URL = 'http://nexus:8081'
    }

    stages { ... }    // All your stages go here
    post { ... }      // Runs AFTER all stages finish
}

Stage: Checkout

stage('Checkout') {
    steps {
        checkout scm   // 'scm' = the Source Control Manager
    }                  // configured in Jenkins job settings
}

checkout scm clones your Git repo into the workspace. scm is a built-in variable Jenkins fills automatically.

Stage: Build

stage('Build') {
    steps {
        sh 'mvn clean package -DskipTests'
        //                  ^ skip tests here (run them separately)
    }
}

sh runs a shell command. -DskipTests skips tests during packaging so the build is fast.

Stage: Test

stage('Test') {
    steps {
        sh 'mvn test'
    }
    post {
        always {   // Runs whether test passed or failed
            junit '**/target/surefire-reports/*.xml'
            // Parses test reports → Jenkins shows pass/fail graph
        }
    }
}

Stage: SonarQube Analysis

stage('SonarQube Analysis') {
    steps {
        withSonarQubeEnv('SonarQube') {
        // ^ Name of your SonarQube server in Jenkins config
            sh 'mvn sonar:sonar'
        }
    }
}

withSonarQubeEnv() injects the SonarQube server URL and token so Maven can connect.

Stage: Quality Gate

stage('Quality Gate') {
    steps {
        timeout(time: 5, unit: 'MINUTES') {
            waitForQualityGate abortPipeline: true
            // abortPipeline: true = stop if gate fails
        }
    }
}

Stage: Publish to Nexus

stage('Publish to Nexus') {
    steps {
        withCredentials([usernamePassword(
            credentialsId: 'nexus-credentials',
            usernameVariable: 'NEXUS_USER',
            passwordVariable: 'NEXUS_PASS'
        )]) {
            script {
                writeFile file: 'maven-settings.xml', text: """..."""
            }
            sh "mvn deploy -DskipTests -s maven-settings.xml"
        }
    }
    post {
        always {
            sh 'rm -f maven-settings.xml'  // Never leave credentials on disk
        }
    }
}

Stage: Deploy

stage('Deploy') {
    steps {
        sh '''
            pkill -f app.jar || true   # Kill old process; || true = don't fail
            nohup java -jar target/app.jar > /home/ubuntu/app.log 2>&1 &
            # nohup = run even after shell exits, & = background
        '''
    }
}

Global Post Block

post {
    success { echo 'Build and deploy successful!' }
    failure { echo 'Build failed!' }
    // Other: always, unstable, aborted, changed
}
04
Environment Variables
▼

Built-in Jenkins Variables

env.BUILD_NUMBER    // "42"
env.BUILD_URL       // "http://jenkins/job/myapp/42/"
env.JOB_NAME        // "myapp"
env.BRANCH_NAME     // "main"
env.WORKSPACE       // "/var/jenkins_home/workspace/myapp"
env.GIT_COMMIT      // full git SHA
env.GIT_BRANCH      // "origin/main"

Defining Your Own

environment {
    NEXUS_URL = 'http://nexus:8081'          // static
    VERSION   = sh(returnStdout: true,        // dynamic
                   script: 'git describe --tags').trim()
    APP_NAME  = "${JOB_NAME}-${BUILD_NUMBER}"  // interpolation
}

Accessing Variables

sh "echo ${env.NEXUS_URL}"   // Groovy interpolation (double quotes)
sh 'echo $NEXUS_URL'         // Shell variable (single quotes)
Important: Use single quotes in sh when you don't want Groovy to interpolate. Use double quotes when you need Jenkins env vars inside the command.
05
Agents & Nodes
▼

agent any

agent any  // Pick any available agent

agent none (stage-level agents)

pipeline {
    agent none   // No global agent
    stages {
        stage('Build') {
            agent { label 'maven' }   // Run on agent tagged 'maven'
            steps { sh 'mvn package' }
        }
        stage('Deploy') {
            agent { label 'prod-server' }
            steps { sh './deploy.sh' }
        }
    }
}

agent with Docker

agent {
    docker {
        image 'maven:3.9-eclipse-temurin-17'
        args '-v $HOME/.m2:/root/.m2'  // Cache Maven deps
    }
}
Docker agents give you fully reproducible builds — the same image always runs the same commands, regardless of what's installed on the host.

agent with multiple labels

agent { label 'linux && docker' }  // Must have BOTH labels
06
Credentials Management
▼
Never hardcode secrets. Jenkins has a built-in encrypted credential store.

Types of Credentials

TypeUsage
usernamePasswordUsername + password pairs (your Nexus setup)
stringSingle secret string / API token
sshUserPrivateKeySSH private key
fileA secret file (e.g., kubeconfig)
certificatePKI cert + key

Username + Password (like your Nexus stage)

withCredentials([usernamePassword(
    credentialsId: 'nexus-credentials',
    usernameVariable: 'USER',
    passwordVariable: 'PASS'
)]) {
    sh 'curl -u $USER:$PASS http://nexus/api'
}

Single String (API Token)

withCredentials([string(credentialsId: 'sonar-token', variable: 'TOKEN')]) {
    sh 'curl -H "Authorization: Bearer $TOKEN" ...'
}

SSH Key

withCredentials([sshUserPrivateKey(
    credentialsId: 'deploy-key',
    keyFileVariable: 'KEY_FILE'
)]) {
    sh 'ssh -i $KEY_FILE ubuntu@server ./deploy.sh'
}
Where to store: Jenkins UI → Manage Jenkins → Credentials → System → Global credentials
07
Post Actions & Notifications
▼

All Post Conditions

post {
    always    { }   // Always runs
    success   { }   // Only on success
    failure   { }   // Only on failure
    unstable  { }   // Tests passed but some failed (yellow)
    aborted   { }   // Job was manually cancelled
    changed   { }   // Status changed from previous build
    fixed     { }   // Was failing, now passing
    regression{ }   // Was passing, now failing
    cleanup   { }   // LAST to run, even after other post conditions
}

Email Notification

post {
    failure {
        mail to: 'team@example.com',
             subject: "FAILED: ${JOB_NAME} #${BUILD_NUMBER}",
             body: "Build failed. Check: ${BUILD_URL}"
    }
}

Slack Notification

post {
    success {
        slackSend channel: '#deployments',
                  color: 'good',
                  message: "Deployed ${JOB_NAME} #${BUILD_NUMBER}"
    }
    failure {
        slackSend channel: '#deployments',
                  color: 'danger',
                  message: "FAILED ${JOB_NAME} #${BUILD_NUMBER} - ${BUILD_URL}"
    }
}
08
Parallel Stages
▼

Run stages simultaneously to save time:

stage('Test & Analyze') {
    parallel {
        stage('Unit Tests') {
            steps { sh 'mvn test' }
        }
        stage('Integration Tests') {
            steps { sh 'mvn verify -Pintegration' }
        }
        stage('Security Scan') {
            steps { sh 'mvn dependency-check:check' }
        }
    }
}
All three run at the same time. The parallel block finishes when all branches finish (or any one fails).

Parameters (Manual Input)
▼
pipeline {
    agent any
    parameters {
        string(name: 'VERSION', defaultValue: '1.0.0', description: 'Release version')
        choice(name: 'ENVIRONMENT', choices: ['dev', 'staging', 'prod'], description: 'Target env')
        booleanParam(name: 'SKIP_TESTS', defaultValue: false, description: 'Skip test stage')
        password(name: 'DEPLOY_KEY', defaultValue: '', description: 'Deployment secret')
    }
    stages {
        stage('Build') {
            steps {
                sh "mvn clean package -Dversion=${params.VERSION}"
            }
        }
        stage('Test') {
            when {
                expression { return !params.SKIP_TESTS }
            }
            steps { sh 'mvn test' }
        }
    }
}

When Conditions (Conditional Stages)
▼
stage('Deploy to Prod') {
    when {
        branch 'main'           // Only on main branch
    }
    steps { sh './deploy-prod.sh' }
}

stage('Deploy to Staging') {
    when {
        anyOf {
            branch 'develop'
            branch 'release/*'
        }
    }
    steps { sh './deploy-staging.sh' }
}

stage('Publish') {
    when {
        allOf {
            branch 'main'
            not { changeRequest() }   // Not a PR
        }
    }
    steps { sh 'mvn deploy' }
}

Shared Libraries
▼

When multiple pipelines share common logic, extract it to a Shared Library in a separate Git repo.

Library Structure

jenkins-shared-lib/
├── vars/
│   ├── buildMaven.groovy       # Global variable (callable as function)
│   └── deployApp.groovy
└── src/
    └── com/example/
        └── PipelineUtils.groovy  # Reusable classes

vars/buildMaven.groovy

def call(String version = '') {
    sh "mvn clean package ${version ? '-Dversion=' + version : ''} -DskipTests"
    junit '**/target/surefire-reports/*.xml'
}

Using the Shared Library

@Library('jenkins-shared-lib') _   // Load it

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                buildMaven('1.2.3')   // Call the shared function
            }
        }
    }
}
Register: Manage Jenkins → Configure System → Global Pipeline Libraries
12
Triggers
▼
pipeline {
    agent any
    triggers {
        // Poll SCM every 5 minutes
        pollSCM('H/5 * * * *')

        // Run on a schedule (nightly at 2am)
        cron('H 2 * * *')

        // Trigger when another job finishes
        upstream(upstreamProjects: 'other-job', threshold: hudson.model.Result.SUCCESS)
    }
}

GitHub Webhook (Better than Polling)

  1. Jenkins URL: http://your-jenkins/github-webhook/
  2. GitHub → Repo → Settings → Webhooks → Add webhook
  3. Content type: application/json
  4. Event: "Just the push event"
Webhooks are better — your Git server calls Jenkins when you push. No polling delay, no wasted network checks.

Multibranch Pipeline
▼

The most powerful Jenkins job type. Jenkins automatically discovers branches and PRs in your repo and creates a pipeline for each.

Setup

  1. Jenkins → New Item → Multibranch Pipeline
  2. Branch Sources → Add → GitHub/GitLab → enter repo URL
  3. Jenkins scans all branches, finds Jenkinsfile in each, creates jobs automatically

Branch-Specific Logic

stage('Deploy') {
    when { branch 'main' }
    steps { sh './deploy-prod.sh' }
}

stage('Deploy to Dev') {
    when { branch 'develop' }
    steps { sh './deploy-dev.sh' }
}

Artifacts & Archiving
▼

Archive Build Outputs

post {
    success {
        archiveArtifacts artifacts: 'target/app.jar',
                         fingerprint: true,
                         onlyIfSuccessful: true
    }
}

Stash/Unstash (Pass Files Between Agents)

stage('Build') {
    steps {
        sh 'mvn package -DskipTests'
        stash includes: 'target/app.jar', name: 'app-jar'
    }
}

stage('Deploy') {
    agent { label 'deploy-server' }  // Different agent!
    steps {
        unstash 'app-jar'            // Get the jar from build agent
        sh 'java -jar target/app.jar'
    }
}

Error Handling & Retry
▼
stage('Flaky Test') {
    steps {
        retry(3) {              // Try up to 3 times before failing
            sh 'mvn verify'
        }
    }
}

stage('Build') {
    steps {
        timeout(time: 10, unit: 'MINUTES') {   // Fail if takes >10 min
            sh 'mvn clean package'
        }
    }
}

stage('Optional Step') {
    steps {
        catchError(buildResult: 'SUCCESS', stageResult: 'UNSTABLE') {
            sh './optional-scan.sh'   // Failure marks stage yellow, not red
        }
    }
}
16
Script Block (Advanced Groovy)
▼

When you need logic beyond what Declarative allows, use a script {} block:

stage('Dynamic Deploy') {
    steps {
        script {
            // Full Groovy available here
            def servers = ['server1', 'server2', 'server3']
            for (server in servers) {
                sh "ssh ubuntu@${server} './deploy.sh'"
            }

            // Capture output from shell
            def gitTag = sh(returnStdout: true, script: 'git describe --tags').trim()
            echo "Deploying version: ${gitTag}"

            // Conditional logic
            if (env.BRANCH_NAME == 'main') {
                sh './notify-team.sh'
            }
        }
    }
}

Docker in Jenkins
▼

Build & Push Docker Image

stage('Docker Build & Push') {
    steps {
        withCredentials([usernamePassword(
            credentialsId: 'dockerhub-creds',
            usernameVariable: 'DOCKER_USER',
            passwordVariable: 'DOCKER_PASS'
        )]) {
            sh """
                docker login -u $DOCKER_USER -p $DOCKER_PASS
                docker build -t myapp:${BUILD_NUMBER} .
                docker tag myapp:${BUILD_NUMBER} myuser/myapp:latest
                docker push myuser/myapp:latest
            """
        }
    }
}

Docker Pipeline Plugin

stage('Build Image') {
    steps {
        script {
            def image = docker.build("myapp:${BUILD_NUMBER}")
            docker.withRegistry('https://registry.example.com', 'registry-creds') {
                image.push()
                image.push('latest')
            }
        }
    }
}

Complete Enhanced Pipeline (Your Project Upgraded)
▼

Your current Jenkinsfile upgraded with everything from this course:

pipeline {
    agent any

    environment {
        NEXUS_URL    = 'http://nexus:8081'
        APP_VERSION  = "${BUILD_NUMBER}"
    }

    triggers {
        pollSCM('H/5 * * * *')
    }

    parameters {
        choice(name: 'DEPLOY_ENV', choices: ['dev', 'staging', 'prod'],
               description: 'Deployment target')
        booleanParam(name: 'SKIP_SONAR', defaultValue: false,
                     description: 'Skip SonarQube (for hotfixes)')
    }

    stages {
        stage('Checkout') {
            steps { checkout scm }
        }

        stage('Build') {
            steps {
                timeout(time: 10, unit: 'MINUTES') {
                    sh 'mvn clean package -DskipTests'
                }
            }
            post {
                success {
                    archiveArtifacts artifacts: 'target/app.jar', fingerprint: true
                    stash includes: 'target/app.jar', name: 'app-jar'
                }
            }
        }

        stage('Test') {
            steps { sh 'mvn test' }
            post {
                always { junit '**/target/surefire-reports/*.xml' }
            }
        }

        stage('Code Quality') {
            when {
                expression { return !params.SKIP_SONAR }
            }
            parallel {
                stage('SonarQube Analysis') {
                    steps {
                        withSonarQubeEnv('SonarQube') {
                            sh 'mvn sonar:sonar'
                        }
                    }
                }
                stage('Dependency Check') {
                    steps {
                        retry(2) { sh 'mvn dependency-check:check' }
                    }
                }
            }
        }

        stage('Quality Gate') {
            when {
                expression { return !params.SKIP_SONAR }
            }
            steps {
                timeout(time: 5, unit: 'MINUTES') {
                    waitForQualityGate abortPipeline: true
                }
            }
        }

        stage('Publish to Nexus') {
            when { branch 'main' }
            steps {
                withCredentials([usernamePassword(
                    credentialsId: 'nexus-credentials',
                    usernameVariable: 'NEXUS_USER',
                    passwordVariable: 'NEXUS_PASS'
                )]) {
                    script {
                        writeFile file: 'maven-settings.xml', text: """<settings>...</settings>"""
                    }
                    sh "mvn deploy -DskipTests -s maven-settings.xml -Denv.NEXUS_URL=${NEXUS_URL}"
                }
            }
            post {
                always { sh 'rm -f maven-settings.xml' }
            }
        }

        stage('Deploy') {
            steps {
                sh """
                    pkill -f app.jar || true
                    nohup java -jar target/app.jar > /home/ubuntu/app.log 2>&1 &
                    echo "App started on ${params.DEPLOY_ENV}"
                """
            }
        }
    }

    post {
        success {
            echo "Build #${BUILD_NUMBER} deployed to ${params.DEPLOY_ENV}"
        }
        failure {
            mail to: 'team@example.com',
                 subject: "FAILED: ${JOB_NAME} #${BUILD_NUMBER}",
                 body: "Check build at: ${BUILD_URL}"
        }
        always {
            cleanWs()   // Clean workspace after every build
        }
    }
}

Troubleshooting Common Issues
▼

Pipeline Not Triggering

  • Check: Webhook delivered? (GitHub → Repo → Settings → Webhooks → Recent Deliveries)
  • Check: Jenkins firewall — can GitHub reach your server?
  • Workaround: Use pollSCM as backup

withCredentials Not Masking Password

  • The variable MUST be accessed via $VAR in shell (not ${VAR} in Groovy string)
  • Use single-quote strings for sh when possible

waitForQualityGate Times Out

  • SonarQube webhook not configured
  • Fix: SonarQube → Administration → Configuration → Webhooks → Add http://jenkins:8080/sonarqube-webhook/

nohup Process Dies After Pipeline

Jenkins kills all child processes when a stage ends.
# Fix: set this env var before nohup
JENKINS_NODE_COOKIE=dontKillMe nohup java -jar app.jar &

Maven Build Fails with "No settings.xml"

  • Your pipeline creates it dynamically — check the writeFile step succeeded
  • Ensure the post { always { rm -f maven-settings.xml } } didn't run before mvn deploy
